# Football Club Site Template - Toman Mariners FC

**Note: This application is a comprehensive template for a professional football club website. "Toman Mariners FC" is used as a placeholder name throughout the project and can be easily rebranded for any team.**

This is the official web platform for **Toman Mariners FC**, built with Next.js and powered by Genkit for AI-driven match insights. It serves as a hub for fans to check fixtures, meet the squad, buy merchandise, and for club administrators to manage content via the Editor Dashboard.

## Features

- **Match Center**: Real-time updates on latest results and upcoming fixtures.
- **AI Match Insights**: Automated summaries and previews generated by Genkit (Gemini 2.5 Flash).
- **First Team Roster**: Detailed profiles of players and coaching staff.
- **Fan Zone**: Integration for tickets, merchandise, and club partnerships.
- **Editor Dashboard**: A restricted area for managing match data, the squad, and staff.

## Technical Stack

- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS + Shadcn UI
- **AI**: Genkit with Google Generative AI (Gemini)
- **Icons**: Lucide React

## Recommended Database Schema (Firestore/NoSQL)

To persist data from the Editor Dashboard, the following schema is recommended for a NoSQL database like Cloud Firestore:

### 1. `fixtures` Collection
- `id`: string (Document ID)
- `opponent`: string
- `date`: string (ISO format)
- `time`: string
- `venue`: string
- `status`: "upcoming" | "completed"
- `result`: (Optional Object)
  - `marinersScore`: number
  - `opponentScore`: number
  - `goals`: Array of Objects
    - `id`: string
    - `player`: string
    - `minute`: number
    - `team`: "Mariners" | "Opponent"

### 2. `players` Collection
- `id`: string (Document ID)
- `name`: string
- `pos`: string (Primary Position)
- `secondPos`: string (Secondary Position)
- `height`: string
- `imageUrl`: string

### 3. `staff` Collection
- `id`: string (Document ID)
- `name`: string
- `role`: string
- `bio`: string
- `imageUrl`: string

## Authentication Guidance

The `/admin` (Editor Dashboard) route should be protected to prevent unauthorized access.

### Recommended Implementation: Firebase Authentication
1. **Providers**: Enable **Email/Password** and **Google** providers in the Firebase Console.
2. **Middleware**: Use Next.js Middleware to check for a valid session token before allowing access to `/admin`.
3. **Security Rules**: If using Firestore, implement Security Rules to ensure only authenticated users with an "admin" or "editor" role can write to the collections.

```typescript
// Example Middleware Logic
export function middleware(request: NextRequest) {
  const session = request.cookies.get('session');
  if (!session && request.nextUrl.pathname.startsWith('/admin')) {
    return NextResponse.redirect(new URL('/login', request.url));
  }
}
```

## Getting Started

1. Clone the repository.
2. Install dependencies: `npm install`
3. Set up your environment variables in `.env` (Google API Key for Genkit).
4. Run the development server: `npm run dev`
