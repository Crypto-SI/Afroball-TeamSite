# Football Club Site Template - Toman Mariners FC

**Note: This application is a comprehensive template for a professional football club website. "Toman Mariners FC" is used as a placeholder name throughout the project and can be easily rebranded for any team.**

This is the official web platform for **Toman Mariners FC**, built with Next.js and powered by Genkit for AI-driven match insights. It serves as a hub for fans to check fixtures, meet the squad, buy merchandise, and for club administrators to manage content via the Editor Dashboard.

## Features

- **Match Center**: Real-time updates on latest results and upcoming fixtures.
- **AI Match Insights**: Automated summaries and previews generated by Genkit (Gemini 2.5 Flash).
- **First Team Roster**: Detailed profiles of players and coaching staff.
- **Fan Zone**: Integration for tickets, merchandise, and club partnerships.
- **Editor Dashboard**: A protected area for managing match data, the squad, and staff.

## Technical Stack

- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS + Shadcn UI
- **AI**: Genkit with Google Generative AI (Gemini)
- **Icons**: Lucide React
- **Recommended Backend**: Supabase (PostgreSQL + Auth)

## Recommended Database Schema (Supabase/PostgreSQL)

To persist data from the Editor Dashboard, the following relational schema is recommended for Supabase:

### 1. `fixtures` Table
- `id`: uuid (Primary Key)
- `opponent`: text
- `date`: date
- `time`: time
- `venue`: text
- `status`: text (check constraint: 'upcoming', 'completed')
- `mariners_score`: integer (nullable)
- `opponent_score`: integer (nullable)

### 2. `goals` Table
- `id`: uuid (Primary Key)
- `fixture_id`: uuid (Foreign Key -> fixtures.id)
- `player_name`: text
- `minute`: integer
- `team`: text (check constraint: 'Mariners', 'Opponent')

### 3. `players` Table
- `id`: uuid (Primary Key)
- `name`: text
- `pos`: text
- `second_pos`: text
- `height`: text
- `image_url`: text

### 4. `staff` Table
- `id`: uuid (Primary Key)
- `name`: text
- `role`: text
- `bio`: text
- `image_url`: text

## Authentication & Security

The `/admin` (Editor Dashboard) route must be protected to ensure only authorized club officials can modify data.

### 1. Supabase Auth
- Enable **Email/Password** or **OAuth** (Google/GitHub) in the Supabase Dashboard.
- Use the `@supabase/auth-helpers-nextjs` (or the newer `@supabase/ssr` package) to manage sessions.

### 2. Row Level Security (RLS)
Enable RLS on all tables. Create policies that allow:
- **Public**: `SELECT` access to all tables.
- **Authenticated Admins**: `INSERT`, `UPDATE`, and `DELETE` access to all tables.

### 3. Middleware Protection
Implement a Next.js Middleware to verify the user's session before allowing access to `/admin`.

```typescript
// Example Middleware Logic (src/middleware.ts)
import { createMiddlewareClient } from '@supabase/auth-helpers-nextjs'
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export async function middleware(req: NextRequest) {
  const res = NextResponse.next()
  const supabase = createMiddlewareClient({ req, res })
  const { data: { session } } = await supabase.auth.getSession()

  if (!session && req.nextUrl.pathname.startsWith('/admin')) {
    return NextResponse.redirect(new URL('/login', req.url))
  }

  return res
}
```

2. Install dependencies: `npm install`
